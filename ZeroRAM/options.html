Basic Settings

Configure basic tab suspension behavior and options

Auto Suspend

Set how long tabs should be idle before automatically suspending to save memory
Set 0 to never auto suspend.
When switching from browser to other applications, protect the last active tab from being suspended
When enabled, clicking the original title or URL will also reload the suspended tab.

Memory Optimization

Choose how to handle suspended tabs for optimal memory usage
Use browser's built-in tab discarding feature for maximum memory savings, but tab state may not be fully preserved.

Whitelist

Manage websites and domains that will never be suspended

Whitelist Rules

Tabs whose URL or domain matches anything in the list will never be suspended.
Enter one URL or domain per line. Examples: • google.com - entire domain • https://example.com/page - specific URL • localhost - local development

Override global time. Format: Pattern : Minutes
• Example Domain: rule34.xxx : 3 • Example URL (Wildcard): https://*id=* : 2 • Example Regex /^https?:\/\/(?:www\.)?rule34.xxx/ : 0.8

Settings Management

Import Settings

Advanced Settings

Advanced user options

Theme Settings

Choose the appearance theme for the extension pages (popup, options, and suspended page)
The theme applies to the popup, options, and suspended pages. Already-suspended tabs need to be reloaded to pick up the change. Auto mode follows your system's dark/light theme preference.

Suspended Tab Indicator

Choose how a suspended tab is marked in the tab strip
Semi-transparent favicon dims the site's icon as the cue: it keeps the site identity but is subtle on tiny tabs and slightly heavier. Title 💤 prefix adds a sleep emoji before the tab title and keeps the favicon at full opacity: it is lighter, but the cue is hidden on very narrow or pinned tabs that show no title.

Suspended Favicon Fix

Control how the extension tries to restore missing favicons on suspended tabs
When a suspended tab shows no favicon after browser restart, try to reload it briefly and discard again to restore its favicon.
Limit how many tabs are processed each time the background scan runs. This helps avoid long processing when many tabs need fixes.
Prevent infinite retries for tabs that cannot be fixed. Set 0 to disable the limit.

Bulk Suspension Concurrency

Control the speed and performance of suspending multiple tabs at once
How many tabs to suspend in parallel when performing bulk operations (like "Suspend Others" or "Suspend All"). Higher values are faster but consume more CPU/memory temporarily.
